How To Report A Bug Or Make A Strong Suggestion

We welcome and are grateful for all bug reports and strong suggestions.  By strong suggestion we mean a helpful observation that we are doing something very wrong (such as violating coding standards, arranging our CVS archives the wrong way, whatever). 

The only way that you can submit a bug report or strong suggestion (we treat them the same way--the bugs database should be used for both) is to use SourceForge's bug tracking system.  Here are the steps you should follow:

  • You must be a registered SourceForge user.  (If you are not, instructions are here.)
  • Please review the information about how categories and groups are used in the bug tracking system.  This will help you to complete the category and group fields when you submit your bug or strong suggestion.
  • Submit your bug or strong suggestion using SourceForge's bug tracking system for The ESRG Tool Set.

Thanks a lot for your help!
