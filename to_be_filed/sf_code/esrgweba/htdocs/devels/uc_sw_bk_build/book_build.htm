Procedure To Build The ESRG Microcontroller Software Book "A Practitioner's Guide ..." For Release Or Other Purposes

The book build process is somewhat unusual in the following ways:

  • The "main" .tex files are generated on-the-fly by a Tcl/Tk script.  This approach is used because it allows the single-volume version of the book and the multi-volume version of the book to be built from the same LaTeX source code without redundancy of information (much information is stored in the Tcl script itself).
  • The build itself is also orchestrated by the Tcl/Tk script.  The script calls the LaTeX compiler as well as other utilities.  (Unfortunately, at this time, the script assumes that the version of LaTeX is 4allTex and it won't work without modification with other versions of LaTeX.)

The main .tex file for the single-volume work is named svf.tex.

The book build process does present a problem for those using a version of LaTeX other than 4allTeX.
